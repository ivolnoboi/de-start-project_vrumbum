Пояснения к модели данных.
Данные о всех сделках вынесены в отдельную таблицу sales, где:
1) id — первичный ключ
2) price — цена в $ с учётом скидки
3) date — дата, когда сделка была проведена
4) discount — скидка в процентах
5) auto_id — внешний ключ к таблице с автомобилями
6) client_id — внешний ключ к таблице с клиентами

Автомобили вынесены в отдельную таблицу autos как отдельная сущность.
Данная таблица связана связью "один ко многим с обязательной связью" с таблицей sales посредством первичного ключа id таблицы autos и внешнего ключа auto_id таблицы sales.
В каждой сделке должен быть обязательно один и только один автомобиль, при этом одна и та же марка автомобиля может принимать участие в различных сделках.
Атрибуты данной таблицы:
1) id — первичный ключ
2) brand_name — марка автомобиля
3) model_name — модель автомобиля
4) gasoline_consumption — потребление топлива на 100 км
5) country_id — внешний ключ к таблице со странами-производителями

Страны-производители находятся в таблице countries и соединяются связью "один ко многим с необязательной связью" с таблицей autos посредством первичного ключа id таблицы countries и внешнего ключа country_id таблицы autos.
У автомобиля может быть либо указана одна страна-производитель, либо не указано ни одной страны. При этом в одной стране может производиться несколько марок автомобилей. 
1) id — первичный ключ
2) name — название страны

Clients — таблица для сущности клиент. Связана с таблицей sales связью "один ко многим с обязательной связью" с использованием первичного ключа id таблицы clients и внешнего ключа client_id таблицы sales.
У одной сделки может быть только один и обязательно один покупатель, и при этом один клиент может купить несколько автомобилей.
Содержит следующие атрибуты
1) id — первичный ключ
2) name — ФИО покупателя
3) phone — номер телефона

У каждого автомобиля есть определенный цвет. И как одна и та же модель автомобиля может иметь разные цвета, так и один цвет может принадлежать разным моделям автомобилей. В связи с этим возникает необходимость использования связи "многие ко многим", что реализуется через использование промежуточной таблицы autos_colors с использованием внешних ключей auto_id и color_id, ссылающихся на таблицы autos и colors соответсвенно.
Внутри этой таблицы всего два поля (auto_id и color_id), которые вместе являются составным первичным ключом.
Таблица colors описывает возможные цвета автомобилей и содержит поля:
1) id — первичный ключ
2) color — цвет
